## Problem Statement

`reason/examples/finance_example.py --live` flags all 6 transactions including the
3 legitimate ones. The live node returns a unit-normalized pattern
`[0.416727, 0.451045, 0.431435, 0.348089, 0.387311, 0.406921]` with thresholds
`{high: 0.82, moderate: 0.62, minimum: 0.42}`.

The pattern is all-positive and unit-normalized — cosine similarity of any
all-positive feature vector against it returns 0.7–0.99. All legitimate transactions
(which have mid-range feature values like 0.12–0.45) score 0.72–0.88, which exceeds
`moderate_confidence = 0.62`. Everything gets flagged.

The simulated path (no `--live`) already works correctly — it uses a raw unnormalized
pattern `[0.88, 0.82, 0.83, 0.89, 0.85, 0.80]` with `high=0.84`. Legitimate
transactions score 0.71–0.75, below the threshold. Fraud scores 0.93–0.99.

## What Makes This Cool

After fix: `--live` output matches the intent of the demo. Legitimate transactions
score ~0.72 → CLEAR. Fraud transactions score ~0.96 → FLAGGED. The example becomes
a correct working demonstration of the protocol.

## Root Cause

The live artifact's thresholds (`high=0.82`) were calibrated before the pattern was
unit-normalized. After normalization, the cosine space compressed and the separation
between fraud and legitimate moved up — thresholds should be ~0.95/0.90/0.85.

## Premises

1. The fix is local to `finance_example.py` — no live artifact re-seeding required now.
2. Calibrated thresholds for the unit-norm pattern: `high=0.97, moderate=0.92, minimum=0.85`.
   Rationale: legitimate transactions (low feature values) score ~0.72–0.85 against the
   unit-norm centroid; fraud (high feature values, 0.75+) score ~0.93–0.99.
3. The override is applied immediately after `client.resolve()` in the live branch,
   before the transaction loop.
4. A comment documents the miscalibration and links to the fix so it's not a mystery.
5. Eventually the live artifact should be re-seeded with corrected thresholds — this
   fix is a temporary shim until that happens.

## Approaches Considered

### Approach A: Local override in --live branch (CHOSEN)
After `artifact = client.resolve(...)`, override `artifact.thresholds` fields.
- Effort: XS | Risk: Zero
- Pros: immediate fix, no network side effects, demo works today
- Cons: thresholds in the live artifact remain wrong; any other code using the live
  artifact also gets wrong thresholds (until re-seed)

### Approach B: --recalibrate flag
Re-seeds the live artifact at `xport.astrognosy.com` with corrected thresholds.
- Effort: S | Risk: Med (changes live state)
- Cons: requires Xport node write access, would invalidate current audit hash,
  changes behavior for all consumers of that URI

## Recommended Approach

**Approach A** — local override with comment. Zero risk, fixes the demo today.
Re-seeding the live artifact is a separate task tracked in next steps.

## Implementation

File: `reason/examples/finance_example.py`

In the `if use_live_node:` block, after the successful `client.resolve()` call,
add before the transactions loop:

```python
# THRESHOLD CALIBRATION SHIM
# The live artifact returns thresholds calibrated before unit-normalization.
# Unit-normalized all-positive patterns compress cosine space upward — legitimate
# transactions score 0.72-0.85 against this centroid, above moderate_confidence=0.62.
# Override with thresholds calibrated for unit-norm space until artifact is re-seeded.
# TODO: re-seed reason://finance/fraud/anomaly-detection with high=0.97, moderate=0.92, minimum=0.85
class _CalibratedThresholds:
    high_confidence = 0.97
    moderate_confidence = 0.92
    minimum_signal = 0.85
    def classify(self, score):
        if score >= self.high_confidence:
            return "high_confidence"
        elif score >= self.moderate_confidence:
            return "moderate_confidence"
        elif score >= self.minimum_signal:
            return "minimum_signal"
        else:
            return "no_signal"
thresholds = _CalibratedThresholds()
```

Note: `pattern` is already set from `artifact.pattern` before this block. Only
`thresholds` is overridden.
