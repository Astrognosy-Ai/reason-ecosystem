## Problem Statement

`reason-py` (pip install reason-py) has a working Python client but a useless CLI.
Running `reason-py` prints a logo and help text. There are no subcommands.
To resolve an artifact or compare features you must open a Python interactive shell,
import ReasonClient, and type multi-line code. This is slow, context-switching, and
painful for demos and quick development checks.

The underlying methods (`client.resolve()`, `client.compare()`) are complete and
working. The gap is purely CLI wiring.

## What Makes This Cool

One command from any terminal: `reason-py compare reason://finance/fraud/anomaly-detection 0.92 0.88 0.79 0.84`
— resolves the artifact from the live node, computes cosine similarity, prints similarity
score + threshold tier + FLAGGED/CLEAR decision. Zero Python boilerplate.

## Constraints

- Must work with `py -3.8` on Windows (no f-strings beyond 3.6, no walrus operator)
- No new dependencies — stdlib `argparse` only
- Must not break the existing `reason-py` (no-args) behavior — logo + help still prints
- `__main__.py` is the only file touched — no new modules

## Premises

1. `client.resolve(uri)` and `client.compare(features, artifact)` in `client.py` are
   complete and correct — CLI just needs argparse wiring.
2. The `compare` subcommand should resolve AND compare in one shot (user selected this).
3. Feature values are passed as positional floats: `reason-py compare <uri> f1 f2 f3...`
4. Exit code 0 = success, 1 = flagged (useful for shell scripting), 2 = error.
5. No-args behavior (`reason-py`) must remain unchanged — logo + help.

## Approaches Considered

### Approach A: Inline argparse in __main__.py (CHOSEN)
Add argparse subparser block to `main()`. Two subcommands: `resolve` and `compare`.
- Effort: S | Risk: Low
- Pros: single file touched, minimal diff, consistent with existing pattern
- Cons: `__main__.py` grows from 125 to ~185 lines
- Reuses: existing `HELP` string, existing `main()` function

### Approach B: Extract to cli.py module
Move CLI logic to a new `cli.py`, import from `__main__.py`.
- Effort: S | Risk: Low
- Pros: cleaner separation
- Cons: new file, import chain, more diff for same result

## Recommended Approach

**Approach A** — inline argparse. Minimum diff, no new files. The 60-line addition is
readable and the module stays under 200 lines total.

## Implementation

File: `reason/sdk/reason_py/__main__.py`

Add argparse block to `main()`:

```python
def main():
    import argparse
    import sys

    parser = argparse.ArgumentParser(
        prog="reason-py",
        description="reason:// protocol CLI",
        add_help=False,
    )
    subparsers = parser.add_subparsers(dest="command")

    # resolve subcommand
    resolve_p = subparsers.add_parser("resolve", help="Resolve a reason:// URI")
    resolve_p.add_argument("uri", help="e.g. reason://finance/fraud/anomaly-detection")
    resolve_p.add_argument("--endpoint", default="https://xport.astrognosy.com")
    resolve_p.add_argument("--bypass-cache", action="store_true")

    # compare subcommand
    compare_p = subparsers.add_parser("compare", help="Resolve URI and compare features")
    compare_p.add_argument("uri", help="e.g. reason://finance/fraud/anomaly-detection")
    compare_p.add_argument("features", nargs="+", type=float,
                           help="Feature values: 0.92 0.88 0.79 ...")
    compare_p.add_argument("--endpoint", default="https://xport.astrognosy.com")

    args = parser.parse_args()

    if args.command is None:
        # Legacy behavior: print logo + help
        print(HELP.format(logo=LOGO, version=__version__))
        return

    from reason_py import ReasonClient
    client = ReasonClient(endpoint=args.endpoint)

    if args.command == "resolve":
        artifact = client.resolve(args.uri, bypass_cache=args.bypass_cache)
        print("uri:             ", artifact.uri)
        print("score:           ", artifact.score)
        print("agent:           ", artifact.provenance.agent_id)
        print("evidence_count:  ", artifact.metadata.evidence_count)
        print("pattern_dims:    ", len(artifact.pattern))
        print("thresholds:")
        print("  high:          ", artifact.thresholds.high_confidence)
        print("  moderate:      ", artifact.thresholds.moderate_confidence)
        print("  minimum:       ", artifact.thresholds.minimum_signal)

    elif args.command == "compare":
        artifact = client.resolve(args.uri)
        similarity = client.compare(args.features, artifact)
        t = artifact.thresholds
        if similarity >= t.high_confidence:
            tier = "HIGH_CONFIDENCE"
            decision = "FLAGGED"
        elif similarity >= t.moderate_confidence:
            tier = "MODERATE_CONFIDENCE"
            decision = "FLAGGED"
        elif similarity >= t.minimum_signal:
            tier = "MINIMUM_SIGNAL"
            decision = "REVIEW"
        else:
            tier = "NO_SIGNAL"
            decision = "CLEAR"
        print("similarity:  {:.4f}".format(similarity))
        print("tier:        {}".format(tier))
        print("decision:    {}".format(decision))
        if decision in ("FLAGGED", "REVIEW"):
            sys.exit(1)
```

## Open Questions

- Should `resolve` also support `--json` output flag for programmatic use? Deferred.
- Should error messages (bad URI, node unreachable) go to stderr? Yes — standard practice,
  not implemented above. Add `except` blocks that write to `sys.stderr` and `sys.exit(2)`.

## Success Criteria

- `reason-py resolve reason://finance/fraud/anomaly-detection` prints artifact fields
- `reason-py compare reason://finance/fraud/anomaly-detection 0.92 0.88 0.79 0.84 0.91 0.77`
  prints similarity + FLAGGED with exit code 1
- `reason-py compare reason://finance/fraud/anomaly-detection 0.05 0.08 0.03 0.45 0.11 0.19`
  prints similarity + CLEAR with exit code 0
- `reason-py` (no args) still prints logo + help
